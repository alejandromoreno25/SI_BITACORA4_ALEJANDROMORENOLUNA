# **1\. Análisis de Necesidades**

Como bien hemos dado en clase, docker y guacamole son plataformas que nos permiten desarrollar y distribuir aplicaciones pero..¿Qué problema resolvemos exactamente con esto?

Docker, nos permite empaquetar y ejecutar aplicaciones en un entorno aislado, lo que nos permite tener todo centralizado y poder ejecutar múltiples contenedores dentro de un mismo host. \[1\]  
 Además, debido a que la plataforma está basada en un sistema de contenedores de trabajo, nos permite que esas cargas de trabajo sean extremadamente portátiles, lo que quiere decir que podemos ejecutar estos contenedores ya sea en ordenador, portátiles o hasta máquinas virtuales.\[1\]

Esta portabilidad facilita mucho toda la gestión del trabajo y nos ayuda a poder aumentar o disminuir las aplicaciones y los servicios según las necesidades del negocio.

A su vez, guacamole es bastante parecido a docker en ese aspecto ya que nos permite un acceso desde un único lugar, facilita  el control de acceso de usuarios y puede ser accedido desde cualquier dispositivo.\[2\]
